# 痛风就医报告生成

基于用户的痛风管理档案，生成结构化的就医报告，便于与医生沟通。

## 核心任务

整理用户的尿酸监测记录、发作历史、用药情况，生成专业的就医报告和问题清单。

## 报告内容结构

### 1. 基本信息摘要

- 患者基本信息
- 痛风病程（首次发作时间）
- 目前治疗方案
- 本次就诊目的

### 5. 重点关注问题

基于数据分析列出需要与医生讨论的问题：

**尿酸控制**
- [ ] 当前尿酸是否达标？
- [ ] 是否需要调整降尿酸药物？
- [ ] 何时可以复查？

**发作控制**
- [ ] 发作频繁是否需要预防性用药？
- [ ] 降尿酸治疗期间的发作如何处理？

**用药问题**
- [ ] 当前药物是否适合我？
- [ ] 是否需要更换药物？
- [ ] 有哪些副作用需要警惕？

**生活方式**
- [ ] 饮食方面还有什么建议？
- [ ] 需要限制哪些活动？

## 不同科室的就诊重点

### 风湿免疫科
- 重点关注：痛风诊断、治疗方案、长期管理
- 常见问题：药物选择、剂量调整、复查频率

### 内分泌科
- 重点关注：代谢综合征相关（血糖、血脂、血压）
- 常见问题：合并症管理、综合治疗

### 肾内科
- 重点关注：肾功能评估、痛风性肾病
- 常见问题：肾功能不全时的用药调整

### 营养科
- 重点关注：个性化饮食方案
- 常见问题：食物选择、膳食搭配

## 触发场景

- 用户准备去看痛风/风湿科
- 用户询问"看病要准备什么"
- 用户需要整理就医资料
- 用户看完病回来需要更新记录

## 注意事项

- 就医报告应信息准确，数据真实
- 问题清单要具体、有针对性
- 就医后及时更新档案信息
- 报告内容仅供就医参考，不替代医生诊断
